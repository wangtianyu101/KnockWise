MCP 协议的 Client/Server 怎么通信？
🤖
MCP 用 JSON-RPC over stdio / HTTP / SSE 三种传输方式。核心是三类对象：
· Resources：只读数据源（文件、数据库行）
· Tools：可执行函数（调 API、查 DB）
· Prompts：可复用 prompt 模板

💡 参考你最近答错的 q_mcp_001 — 你当时没答出 Resources/Tools 区别，建议重看那题。
👤
那 SSE 模式下服务端主动推送怎么实现？
🤖
SSE（Server-Sent Events）是 keep-alive HTTP 连接，服务端通过事件流（event-stream）格式持续推送消息。
客户端用 EventSource 接收，单向（服务端 → 客户端）。
比 WebSocket 简单，但只能推不能双向。

💡 MCP 用 SSE 是因为大部分场景只是服务端发通知给客户端（如工具列表变更）。

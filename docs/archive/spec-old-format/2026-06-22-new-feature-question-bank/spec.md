## 一、设计目标

### 1.1 解决什么问题

之前的 3 个缺陷 + 1 个新发现：
1. 没有"学"环节
2. 没有"沉淀"环节
3. 没有"循环"
4. **新发现**：模块互相混合（面试报告里有"建议重做的题"、日报里有"今日推荐题"）—— 每个模块应该是独立闭环

### 1.2 这次要做到什么

**4 大完全独立模块**：

| # | 模块 | 路径 | 内容 |
|---|---|---|---|
| ① | **面试** | `/interview/*` | 实战 AI 追问 + 报告（独立） |
| ② | **学习复习** | `/learn/*` `/review/*` `/qa/*` | 1 模块 3 tab：学 + 复习 + 问答（独立） |
| ③ | **知识库** | `/knowledge` | Obsidian 集成（独立） |

**核心原则**：每个模块**完全独立**，不跨模块混合数据。

### 1.3 目标用户

| 人群 | 经验 | 痛点 |
|---|---|---|
| **目标用户 A：AI Agent 求职准备者** | 1-5 年 | 想系统刷题 + 实战面试，但题源散落 |
| **目标用户 B：已入职持续学习者** | 在职 | 学了新东西容易忘，需要"复习" + "AI 问答" |

### 2.2 模块独立性原则

**绝对禁止**的跨模块混合：

| 禁止 | 原因 |
|---|---|
| 面试报告里显示"建议重做的题" | 这是学习复习的事 |
| 面试报告里显示"相关 Obsidian 笔记" | 这是知识库的事 |
| 学习复习里显示"面试实战表现" | 面试有自己独立的报告 |
| 知识库里"建议写笔记关联到某道题" | 题库的事跟知识库分开 |
| Dashboard 跨模块推荐 | 每个模块独立入口 |

**允许的**全局数据：Profile 沉淀 + 全局导航 + 统一登录。

```

| 题目去重 | `(user_id, question_id)` UNIQUE |
| 答题数据 | 只更新 `question_progress` 表，**不更新** `interview` / `report` |

## 五、关键用户旅程

### 5.1 流 A：日常学习（学习复习 · 学 tab）

```
打开 /learn（默认 tab=学）
        │
        ▼
看顶部统计（已做 18 / 掌握 5 / 学习中 8）
        │
        ▼
看今日推荐 3 题（弱项 1 + 复习 1 + 未练 1）
        │
        ▼
点开第 1 道 → /learn/[qid]
        │
        ▼
读题 → 写答案 → 提交 → AI 评分（4/5）+ 反馈
        │
        ▼
SRS 自动重算（下次复习 3 天后）
切掌握度为"学习中"（如果第 2 次答对则自动"已掌握"）
        │
        ▼
点"下一题" → 拉 SRS 队列 / 今日推荐
        │
        ▼
做完 3 题 → 回到 /learn → 顶部统计 +1
```

### 5.2 流 B：复习（学习复习 · 复习 tab）

```
打开 /review
        │
        ▼
看"今日到期 5 道 · 预计 20 分钟"卡片
        │
        ▼
点"开始今日复习" → 进入 /learn/[qid] 模式（带复习进度条）
        │
        ▼
依次答 5 道题
        │
        ▼
每答完一道，next_review_at 按 SRS 重算
        │
        ▼
5 道全做完 → 回到 /review → "今日到期 0 道"
        │
        ▼
看"复习统计"：本周完成 8 / 到期 10 = 80% 完成率
```

### 5.3 流 C：AI 自由问答（学习复习 · 问答 tab）

```
打开 /qa
        │
        ▼
看左侧历史会话列表
        │
        ▼
点"+ 新会话" 或 复用旧会话
        │
        ▼
输入问题："MCP 协议 Client/Server 怎么通信？"
        │
        ▼
AI 调用 LLM，上下文注入：
  - 用户最近答过的 20 道题（question_progress 表）
  - 用户最近写的 5 篇笔记（如果知识库有数据）
  - 用户错题本
        │
        ▼
AI 回答 + 引用用户学过的内容
        │
        ▼
继续追问（多轮对话）
        │
        ▼
退出 → 历史会话保存到 DB
```

        │
        ▼

        │

```

## 六、Profile 沉淀（统一容器，每个模块独立统计）

### 6.1 沉淀字段（不分模块，共用 Profile）

| 字段 | 用途 | 写入方 |
|---|---|---|
| `interview_stats` | 面试统计（场数 / 均分 / 趋势） | 面试模块完成时 |
| `learn_stats` | 学习复习统计（学/复习/问答次数） | 学习复习完成时 |
| `knowledge_stats` | 知识库统计（笔记数 / 字数） | 知识库浏览时（不主动写入） |
| `last_active_at` | 最近活动 | 任意模块 |

> **关键**：每个模块写自己的 `*_stats`，**不**跨模块写。比如面试模块**不**写 `learn_stats`。

## 七、边缘场景

| 场景 | 行为 |
|---|---|
| 用户在面试中答对某题 | ❌ 不更新 `question_progress`（面试独立） |
| 面试报告中"建议重做的题" | ❌ 不显示（用户决定：完全独立） |
| 知识库里"建议关联题目" | ❌ 不显示 |
| 问答 tab 里"调用知识库" | ❌ 不调用（只用学习复习自己的数据） |
| 跨模块搜索（"找关于 MCP 的所有内容"） | ❌ 不做（每个模块独立搜索） |

## 八、成功指标

| 指标 | 目标 | 测量方式 |
|---|---|---|
| **学习复习 DAU** | > 30% | 每日有学/复习/问答行为用户 / 总 DAU |
| **复习完成率** | > 70% | 今日到期题中已答的比例 |
| **问答使用率** | > 20% | 每日有问答行为的 DAU 比例 |
| **4 模块独立使用** | 每个模块 MAU > 10% | 各模块月度活跃用户 |
| **跨模块零污染** | 0 跨模块数据写入 | 代码审计 |

## 九、不在本设计内

- 资源管理（文章/视频/书）
- 知识图谱的高级可视化（节点颜色/大小/聚类）
- 激励机制（streak / 等级 / 徽章）
- 移动端
- 跨模块推荐（Dashboard 跨模块卡片）
- 任何"建议从 X 模块做 Y"的引导

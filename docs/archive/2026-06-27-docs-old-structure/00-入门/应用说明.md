# 应用说明

> 一个真正会「追问」的 AI Agent 工程师语音面试 Agent。
> 不是题库生成器，是模拟真实面试官的 AI。

---

## 一、项目简介

KnockWise（项目目录名 `Intervue`，UI 品牌名 `KnockWise`）是面向 1–5 年 AI Agent 工程师求职者的模拟面试平台。核心差异化是**追问引擎** —— 不是静态题库，而是根据用户回答动态追问的 AI 面试官。

三个核心模块：
1. **面试练习** — AI 追问引擎 + 实时语音 + 报告
2. **知识管理** — Obsidian vault 集成，全文检索 + 知识图谱
3. **信息推送** — AI 日报 / 周报 + 代码统计

## 五、启动方式

本项目提供两种启动方式：**本机模式（推荐）** 和 **Docker 模式**。Docker 模式因为 ghcr.io / Docker Hub 在国内拉镜像经常超时，目前**实测走不通**，建议用本机模式。

### 5.1 本机模式（推荐）

#### 第一步：准备 MySQL

```bash
# 用 Homebrew 装（macOS）
brew install mysql
brew services start mysql

# 创建库和用户
mysql -u root -p
> CREATE DATABASE knockwise CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
> CREATE USER 'knockwise'@'localhost' IDENTIFIED BY 'knockwise';
> GRANT ALL ON knockwise.* TO 'knockwise'@'localhost';
> FLUSH PRIVILEGES;
```

如果是阿里云 RDS，把 `localhost` 换成 RDS 内网地址即可。

#### 第二步：装 LiveKit（仅本机语音需要）

```bash
brew install livekit
# 配置文件：./livekit.yaml
livekit-server --config ./livekit.yaml --node-ip 127.0.0.1
```

不连 LiveKit 也能用 —— 文字面试路径不依赖它。

#### 第三步：启动后端

```bash
cd backend
./.venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000 --env-file .env.local
```

启动时会自动：
- `init_db()` 创建缺失的表
- `_run_migrations()` 跑轻量级 ALTER（自动加 `is_favorite` / `deleted_at` / `summary` / `overall_score` 等列）
- 预热 STT 模型（faster-whisper tiny，约 75 MB）

#### 第四步：启动前端

```bash
cd frontend
npm install   # 第一次跑
npm run dev
```

打开 http://localhost:3000

#### 第五步：登录

`/` 登录页有 **Dev Login** 按钮（无密码直接进，方便本地调试）。生产环境请用邮箱注册或 GitHub OAuth。

### 5.2 Docker 模式（不一定可用）

> ⚠️ 实测在国内网络下 `docker pull mysql:8.4` 和 `ghcr.io/collabora/whisperlive` 经常 timeout。Docker Hub 配置镜像加速器后可解。

```bash
# 1. 配置 Docker Hub 镜像加速器（写到 ~/.docker/daemon.json）
#    例如阿里云：https://<你的ID>.mirror.aliyuncs.com

# 2. 拉起全部服务
docker-compose up -d

# 3. 健康检查
curl http://localhost:8000/api/health
```

服务列表（Docker 模式）：
| 服务 | 端口 |
|------|------|
| mysql | 3306 |
| whisper-live | 9090 |
| livekit | 7880/7881/7882 |
| backend | 8000 |
| frontend | 3000 |

### 6.3 端到端调用样例

```bash
# 1. dev 登录（跳过密码）
TOKEN=$(curl -s "http://localhost:8000/api/auth/dev-login?username=test" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

# 2. 启动面试
IVID=$(curl -s -X POST http://localhost:8000/api/interviews \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"round":"round1","style":"standard"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

# 3. 拿题 → 回答 → 拿下一题（循环）
Q=$(curl -s -X POST "http://localhost:8000/api/interviews/$IVID/next-question" -H "Authorization: Bearer $TOKEN")
REC=$(echo "$Q" | python3 -c "import sys,json; print(json.load(sys.stdin)['record_id'])")
curl -s -X POST "http://localhost:8000/api/interviews/records/$REC/answer" \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"user_answer":"ReAct = Reasoning + Acting 循环","time_spent":15}'

# 4. 结束面试
curl -s -X POST "http://localhost:8000/api/interviews/$IVID/complete" -H "Authorization: Bearer $TOKEN"

# 5. 生成报告
curl -s -X POST "http://localhost:8000/api/reports/interview/$IVID" -H "Authorization: Bearer $TOKEN"
```

## 八、故障排查

### 8.1 `Failed to fetch`（前端报错）
后端没起 / 端口被改。检查：
```bash
curl http://localhost:8000/api/health
lsof -nP -iTCP:8000 -sTCP:LISTEN
```

### 8.2 `Unknown column 'xxx' in 'field list'`
数据库旧表缺新列。`core/database.py` 的 `_MIGRATIONS` 应在启动时自动 ALTER。如果失败，看后端启动日志 `migration xxx failed: ...`。

### 8.3 面试记录突然暴涨
曾经存在的 bug：点"开始面试"会因 React StrictMode 双触发 + room 页忽略 `?id=` 重复创建。已在 `api/interview.py:start_interview` 加去重，在 `pages/interview/room.tsx` 和 `pages/interview.tsx` 加守卫。清理脏数据：
```sql
UPDATE interviews SET deleted_at = NOW() WHERE status='in_progress' AND deleted_at IS NULL;
```

### 8.4 报告是全 3 的 stub
旧版问题，已修。`api/report.py` 现在调真 `ReportAgent.generate_report()`。

### 8.5 简历上传 OCR 失败
`DASHSCOPE_API_KEY` 没配 / 不在阿里云网络。检查 `backend/.env.local`，非扫描件 PDF 走 pypdf 提取，不需要 OCR。

### 8.6 LiveKit 房间连不上
`livekit.yaml` 里 `node_ip` 是 192.168.1.20，本机测试时改 127.0.0.1（启动时加 `--node-ip 127.0.0.1`）。

### 8.7 知识库空
`~/Obsidian/coding/` 目录不存在或没 `.md` 文件。`services/obsidian_service.py:VAULT_ROOT` 是硬编码 `Path.home() / "Obsidian" / "coding"`，要改路径改这一行。

## 一、设计目标

### 1.1 核心问题

当前系统存在三个结构性缺陷：

1. **没有"学"环节** —— 面试模块是"练"，知识模块是"看"，但用户**没有系统刷题**的入口
2. **没有"沉淀"环节** —— 每场面试结束就结束了，盲点 / 掌握度 / 成长曲线**没沉淀到用户画像**
3. **没有"循环"** —— 三模块是 3 个孤岛，AI 推荐只做了一点点（盲点 → 笔记），没有形成 `学 → 练 → 反思 → 调整` 的闭环

### 1.2 设计目标

构建一个 **4 模块 × 3 层记忆 × 1 张画像** 的闭环系统：

- **4 模块**：题库练习（新增）/ 面试练习 / 知识管理 / 信息推送
- **3 层记忆**：短时（当前面试） / 中时（近期面试） / 长时（RAG 题库）
- **1 张画像**：Profile 沉淀所有数据

### 1.3 与认知科学对应

| 系统层级 | 认知科学 | 类比 | 物理存储 |
|---|---|---|---|
| Layer 1 短时 | 工作记忆（Working Memory） | 桌子上正在写的那张草稿 | `interviews.state_snapshot` JSON |
| Layer 2 中时 | 短时记忆（Short-term） | 最近的笔记本 | `interviews` + `question_records` 行（最近 3-5 场） |
| Layer 3 长时 | 长时记忆（Long-term） | 脑子里的知识 | 向量库 + `QuestionProgress` 表 |
| Profile 沉淀 | 自我概念（Self-concept） | "我是谁、我擅长什么" | `profiles` 表 |

## 三、3 层记忆详细设计

### 3.1 Layer 1：短时记忆（当前面试）

**职责**：保存"正在发生"的面试状态，包括：
- 当前题目、追问链
- 用户正在输入的答案（草稿）
- 上一题的回答 + 评分 + 追问动作
- VAD / 打断状态（语音模式）

- 主存储：`interviews.state_snapshot` JSON 列（**已有**）
- 访问入口：`backend/services/interview_service.py:InterviewSessionManager`（**已有**）

### 3.2 Layer 2：中时记忆（近期面试）

**职责**：跨场次的"最近记忆"，给 question_agent / followup_agent / report_agent 提供上下文。

**存储**：
- `interviews` 行（最近 3-5 场，状态 = completed）
- `question_records` 行（这些面试的所有题目）

**容量策略**（滑动窗口）：

```python
# 新增 backend/services/recent_memory.py
RECENT_INTERVIEW_WINDOW = 5      # 保留最近 5 场
RECENT_INTERVIEW_DAYS = 30       # 且 30 天内

async def get_recent_interviews(user_id: str, db: AsyncSession) -> list[Interview]:
    """滑动窗口：最近 5 场 或 30 天内，取并集。"""
    cutoff = datetime.now(timezone.utc) - timedelta(days=RECENT_INTERVIEW_DAYS)
    result = await db.execute(
        select(Interview)
        .where(
            Interview.user_id == user_id,
            Interview.status == "completed",
            Interview.ended_at >= cutoff,
        )
        .order_by(Interview.ended_at.desc())
        .limit(RECENT_INTERVIEW_WINDOW)
    )
    return result.scalars().all()
```

**使用场景**：

| 调用方 | 怎么用 Layer 2 |
|---|---|
| `question_engine.select_next_question` | 排除最近 5 场问过的题（`questions_asked` 已经做了，去重窗口扩大） |
| `followup_engine.determine_action` | 参考最近 3 场"我追问什么最有效"，调整追问风格 |
| `report_agent.generate_report` | 跟最近 3 场对比，识别"反复出现的盲点"vs"已改善的薄弱项" |
| 前端 dashboard | "上次面试你答得最差的是 X，这周重点练 X" |

### 3.3 Layer 3：长时记忆（RAG 题库）

**职责**：跨时间的"永久记忆"，支持语义检索 + 用户掌握度追踪。

**存储**：
- **向量库**：每道题（题面 + topic + sub_topic）embedding → 存入本地向量库
- **关系表**：`QuestionProgress`（用户对每道题的掌握度）

**向量库选型**：

| 选项 | 体积 | 性能 | 部署 | 推荐度 |
|---|---|---|---|---|
| **chroma** | 100KB+ 数据 | 中 | pip 装，本地文件 | ⭐⭐⭐⭐⭐ MVP 首选 |
| faiss-cpu | 库 ~5MB | 快 | pip 装 | ⭐⭐⭐⭐ |
| pgvector | MySQL 扩展 | 中 | 需扩展 | ⭐⭐ 项目已经用 MySQL |
| 阿里云向量检索 | — | — | 云端 | ⭐⭐⭐ 数据出本机 |

**首选 chroma**（轻量、文件式存储、容易调试）。

**Embedding 模型**：

**新增 service**：`backend/services/long_term_memory.py`

```python
"""Layer 3 长时记忆：向量检索 + 掌握度追踪。"""

class LongTermMemory:
    def __init__(self, vector_path: str = "./data/chroma"):
        import chromadb
        self.client = chromadb.PersistentClient(path=vector_path)
        self.collection = self.client.get_or_create_collection(
            name="questions",
            metadata={"hnsw:space": "cosine"},
        )

    def index_question(self, qid: str, text: str, topic: str, metadata: dict):
        """把一道题 embedding 入库（首次启动时全量灌入）。"""
        self.collection.upsert(
            ids=[qid],
            documents=[text],
            metadatas=[{"topic": topic, **metadata}],
        )

    def find_similar(self, query: str, exclude: list[str], k: int = 5) -> list[dict]:
        """语义检索：找相似题但排除已问过的。"""
        results = self.collection.query(
            query_texts=[query],
            n_results=k + len(exclude),  # 多取一些，filter 完剩 k
            where={"topic": {"$ne": "skip_topic"}},
        )
        out = []
        for i, qid in enumerate(results["ids"][0]):
            if qid in exclude:
                continue
            out.append({
                "question_id": qid,
                "text": results["documents"][0][i],
                "score": 1 - results["distances"][0][i],  # cosine distance → similarity
                "metadata": results["metadatas"][0][i],
            })
            if len(out) >= k:
                break
        return out

    def find_for_blindspot(self, blind_spot: str, user_id: str, k: int = 3) -> list[dict]:
        """针对一个盲点，找语义相关的题（用于复习推荐）。"""
        # 同时用盲点文字 + 用户最近答错的题 做扩展查询
        return self.find_similar(blind_spot, exclude=[], k=k)
```

**新增 service**：`backend/services/profile_settlement.py`

```python
async def settle_after_interview(user: User, interview: Interview, db: AsyncSession):
    """每次面试完成调用。轻量：只更新 weak_topics。"""
    profile = await get_or_create_profile(user, db)

    # 聚合本场盲点
    records = await db.execute(
        select(QuestionRecord).where(QuestionRecord.interview_id == interview.id)
    )
    blind_counts = Counter()
    for r in records.scalars().all():
        for spot in (r.blind_spots or []):
            blind_counts[spot] += 1

    # 合并到 profile.weak_topics
    existing = {w["topic"]: w for w in (profile.weak_topics or [])}
    for spot, count in blind_counts.most_common(5):
        if spot in existing:
            existing[spot]["count"] += count
            existing[spot]["last_seen"] = datetime.now(timezone.utc).isoformat()
        else:
            existing[spot] = {
                "topic": spot,
                "count": count,
                "first_seen": datetime.now(timezone.utc).isoformat(),
                "last_seen": datetime.now(timezone.utc).isoformat(),
            }
    profile.weak_topics = list(existing.values())
    profile.last_active_at = datetime.now(timezone.utc)
    await db.commit()


async def settle_weekly(user: User, db: AsyncSession):
    """每周一凌晨调用。深度：重算 learning_trajectory + 降级 Layer 2 到 Layer 3。"""
    profile = await get_or_create_profile(user, db)

    # 全部 completed 面试
    interviews = await get_all_completed_interviews(user.id, db)
    if not interviews:
        return

    scores = [i.overall_score for i in interviews if i.overall_score is not None]
    profile.learning_trajectory = {
        "interview_count": len(interviews),
        "avg_score": round(sum(scores) / len(scores), 2) if scores else None,
        "first_score": scores[-1] if scores else None,
        "latest_score": scores[0] if scores else None,
        "improvement": round(scores[0] - scores[-1], 2) if len(scores) >= 2 else None,
        "score_trend": "up" if len(scores) >= 2 and scores[0] > scores[-1] else "flat",
    }
    await db.commit()

    # 30 天前的 interview：提取摘要到 Layer 3
    cutoff = datetime.now(timezone.utc) - timedelta(days=30)
    old = [i for i in interviews if i.ended_at and i.ended_at < cutoff]
    for iv in old:
        await _archive_to_layer3(user.id, iv, db)
```

```txt
FlagEmbedding            # （可选）bge 官方 Python 包
```

注意：`sentence-transformers` 会拉 torch（~700MB），如果不想这么重，可以直接用 `FlagEmbedding` 单独跑 bge（~50MB 模型权重）。

## 八、实施路径

按依赖排序，建议分 6 个阶段：

| 阶段 | 工作量 | 内容 | 依赖 |
|---|---|---|---|
| **P0** 数据模型 | 0.5 天 | 新增 `QuestionProgress` 表 + 改 `Profile` 字段 + 写 migration | 无 |
| **P1** Layer 3 RAG 基建 | 1-2 天 | 装 chroma + bge-small-zh + `long_term_memory.py` + 50 道种子题入库 | P0 |
| **P2** Profile 沉淀 service | 0.5-1 天 | `profile_settlement.py`（轻量 + 深度） + 触发器接入 | P0 |
| **P3** Layer 2 + API | 1 天 | `recent_memory.py` + 3 个新 endpoint | P0 |
| **P4** 题库练习前端 | 2 天 | `/practice` 列表 + `/practice/[qid]` 答题 + 4 档掌握度 | P0 + P1 |
| **P5** Dashboard + Report 升级 | 1-2 天 | 4 模块卡片 + 成长曲线 + 推荐题 | P0 + P1 + P2 + P3 |
| **P6** "我的日报" + 闭环自动化 | 1-2 天 | 每日日报生成 + launchd 定时 + AI 日报选题 → 题库推荐 | 全部 |

**MVP 最小可用闭环 = P0 + P1 + P2 + P3 + P4，约 5-6 天**。

## 九、风险与权衡

| 风险 | 影响 | 缓解 |
|---|---|---|
| 装 `sentence-transformers` 拉 torch ~700MB | 部署包变大 | 改用 `FlagEmbedding` 单独跑 bge（~50MB） |
| chroma 写入慢 | 题库超过 1000 题时入库耗时 | 批量 `upsert`，后台异步 |
| bge-small-zh 准确度不够 | 误判相似题 | 可热切换到 bge-base-zh 或 OpenAI embedding |
| Layer 2 滑动窗口 SQL 慢 | 面试多了以后 `get_recent_interviews` 全表扫 | 加 `idx_user_ended (user_id, ended_at)` 复合索引 |
| Profile 字段 JSON 化导致查询不便 | 不能 SQL 查 `weak_topics` | 数量小（< 20）时 JSON OK；要强查询可拆出 `profile_topics` 表 |
| 30 天降级任务跑崩 | 老数据没归档 | 加 launchd restart + 失败告警 |
| 闭环只对单人有效 | 多设备同步？ | 项目是单用户，不考虑 |
| 误标记"已掌握" | 用户不再复习 | 标记逻辑要"答对 N 次"而不是单次 |

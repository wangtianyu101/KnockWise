# AI 推送 设计（产品文档 · 独立模块）

> 阶段 4.2 独立设计 · 状态：📋 产品设计
> **核心价值**：让用户用 5-10 分钟/天，跟上 AI 圈最新动态
> **设计目标**：每日 + 每周总结 + **体验优化**（不是链接列表，是"读完"）
>
> **独立性原则**：AI 推送不调用其他模块的 API、不读其他模块的数据表。
> 唯一全局：Profile 里独立的 `digest_stats`（已读 / 收藏 / 阅读时长）。
>
> 配套文档：
> - [`./AI推送-技术设计.md`](./AI推送-技术设计.md) — 数据模型 / RSS / LLM 选题 / 邮件
> - [`./AI推送-页面规划.md`](./AI推送-页面规划.md) — UI 草图 + 流程
> - [`../designs/AI推送-页面设计.html`](../designs/AI推送-页面设计.html) — 可打开的设计图
>
> **替代关系**：本模块**从 `面试题库设计.md` 独立出来**。面试题库不再含 AI 推送内容。

## 一、设计目标

### 1.1 解决什么问题

| 之前 | 现状 |
|---|---|
| 用户每天刷 30+ RSS 源，浪费时间 | 用 LLM 自动选题 + 摘要 |
| 外部资讯和个人学习混在一起 | 完全独立，**不显示学习数据** |
| 没有"读完"的体验（链接列表） | 5-10 分钟读完，有进度感 |
| 推送时机不可控 | 多渠道 + 用户自选 |

### 1.2 这次要做到什么

**5 大核心功能**：

1. **每日日报** — 早上 8 点推送，10 条精选 + LLM 摘要
2. **每周周报** — 周一早上 8 点推送，本周 5 大事件 + 趋势
3. **每月月报** — 每月 1 号推送，本月趋势 + 重磅论文
4. **AI 智能选题** — 基于用户关注标签 + 互动历史（已读 / 收藏 / 跳过）
5. **多渠道推送** — 站内 + 邮件 + macOS notification

### 1.3 体验优化目标（核心）

| 指标 | 目标 |
|---|---|
| **读完率** | > 70%（不是打开率，是真读完） |
| **每日阅读时长** | 5-10 分钟 |
| **收藏率** | > 15%（看到好内容会收藏） |
| **周报打开率** | > 80%（每周必看） |
| **推送不被屏蔽率** | < 5% |

### 1.4 目标用户

| 人群 | 痛点 |
|---|---|
| **目标用户 A：AI 圈从业者** | 每天信息过载，需要过滤 |
| **目标用户 B：技术 leader** | 需要趋势判断（关注什么 / 投什么） |
| **目标用户 C：投资人 / PM** | 需要商业侧 + 论文双轨 |

### 2.2 与其他模块的边界

**AI 推送 不调用任何其他模块**：

| ❌ 不调用 | 说明 |
|---|---|
| /api/learn/* | 不看用户的题库 progress |
| /api/qa/* | 不看用户的问答历史 |
| /api/knowledge/* | 不读 Obsidian 笔记 |
| /api/interview/* | 不看面试报告 |
| /api/profile/settlement | 不调用 settlement |

**AI 推送 只写自己的表**：

- `daily_digest`（日报存档）
- `weekly_digest`（周报存档）
- `monthly_digest`（月报存档）
- `digest_source`（信源配置）
- `digest_subscription`（用户订阅/偏好）
- `digest_read`（阅读记录）
- `digest_bookmark`（收藏）

**唯一全局**：`profiles.digest_stats`（已读 / 收藏 / 阅读时长，由 digest 模块自己写）。

### 2.3 页面路由

```
/push                    # 总览（今日 + 本周 + 本月 tab）
/push/daily/[date]      # 某天日报详情
/push/weekly/[week]      # 某周周报详情
/push/monthly/[month]    # 某月月报详情
/push/sources            # 信源管理
/push/settings            # 推送设置（标签 / 时间 / 渠道 / 频率）
/push/bookmarks          # 我的收藏
```

```
│    [收藏]  [分享]  [🔇 不再推类似]                              │
│                                                              │
│  🏢 商业（2 条）                                               │
│  · [某 AI 公司融资 5 亿]        2 分钟   ⭐ 3.5         │
│    ...                                                          │
│                                                              │
│  📚 论文（2 条）                                               │
│  · [arXiv 论文]                  5 分钟   ⭐ 4.2         │
│    ...                                                          │
│                                                              │
│  🛠 工程（2 条）                                               │
│  · [某框架 1.0 发布]             2 分钟   ⭐ 3.8         │
│    ...                                                          │
│                                                              │
│  💡 观点（1 条）                                                │
│  · [某大 V 评论]                 1 分钟   ⭐ 4.0         │
│    ...                                                          │
│                                                              │
│  ────────────────────────────────────────────────────────    │
│  进度: ████░░░░  4/10 条已读  ·  剩余 3 分钟                    │
│  [全部已读 · 明天 8:00 继续]                                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

#### 关键 UX 规则

- **分类明确**：🔥头条 / 🏢商业 / 📚论文 / 🛠工程 / 💡观点（最多 5 个分类）
- **每条带预估时长**：用户知道这篇要读多久
- **每条带质量分 ⭐**：基于 LLM 评估的 1-5 分
- **每条带操作**：收藏 / 分享 / 不再推类似
- **进度条**：10/10 已读 → "明天 8:00 继续"

#### 摘要质量（核心）

| ❌ 差摘要 | ✅ 好摘要 |
|---|---|
| "OpenAI 发布了 GPT-5" | "OpenAI 发布 GPT-5 多模态版。相比 4o 提升 3 倍上下文（128K→2M token），新增视觉理解。但 30$/月定价引发争议。" |

**摘要 = 核心事实 + 关键数字 + 影响**（3-5 行）

```
│  • Agent 框架白热化（4 款新框架发布）                            │
```

#### 周报 UX 规则

- **5 大事件按影响力排序**（不是按时间）

- **下周展望** 让用户有预期
- **对比上周**按钮：看增长 / 下滑
- **8 分钟读完** 是目标（不是 30 分钟）

## 四、关键 UX 设计

### 4.1 阅读体验（"读完" 而非"打开"）

#### 当前 RSS 体验的问题

- ❌ 30+ 条原始链接，看不完
- ❌ 摘要质量参差
- ❌ 没有进度感
- ❌ 没有回顾机制

#### AI 推送的解决

- ✅ **精选 10 条 / 天**（从 50+ 原始中筛）
- ✅ **LLM 摘要 3-5 行**（核心事实 + 数字 + 影响）
- ✅ **进度条**（N/10 已读）
- ✅ **预估时长**（每条标"3 分钟"）
- ✅ **已读后变灰**（不重复打扰）
- ✅ **第二天自动归档**（但可回看）

### 4.2 推送时机（智能）

| 渠道 | 时机 | 触发 |
|---|---|---|
| 站内 | 用户打开 /push 页面时显示 | 实时 |
| 邮件 | 每天 8:00 准时 | cron |
| macOS notification | 用户主动订阅时 | cron |
| 微信 / 飞书 | 暂不做（P1 阶段） | - |

**关键**：用户可调推送时间（默认 8:00，可改 7:00 / 9:00）。

### 4.3 智能选题（核心算法）

```
RSS 抓取 → 50+ 条原始
    ↓
LLM 选题（输入：用户偏好 + 互动历史 + 50 条原始）
    ↓
10 条精选（按用户兴趣权重）
    ↓
LLM 摘要（每条 3-5 行）
    ↓
按分类排版（🔥 头条 / 🏢 商业 / 📚 论文 / 🛠 工程 / 💡 观点）
    ↓
写数据库 + 触发邮件 + macOS notification
```

**用户偏好** = 关注标签（如 "Agent", "RAG", "国产模型"）+ 互动历史（已读 / 收藏 / 跳过）

### 4.4 信源管理（用户可控）

```
┌─ 信源管理 ────────────────────────────────────────┐
│  ✅ 量子位          大模型     启用 [停用]         │
│  ✅ 36氪            应用       启用 [停用]         │
│  ✅ HuggingFace     论文       启用 [停用]         │
│  ✅ arXiv cs.AI     论文       启用 [停用]         │
│                                                      │
│  [+ 添加自定义 RSS 源]                              │
└──────────────────────────────────────────────────────┘
```

### 4.5 推送设置（用户自选）

```
┌─ 推送设置 ────────────────────────────────────────┐
│  ⏰ 推送时间: [8:00] ▾                              │
│  📅 推送频率: [☑ 每日] [☑ 每周] [☐ 每月]          │
│  📧 邮件推送:  [☑ 启用]                            │
│  🔔 macOS:     [☐ 启用]                            │
│  🏷 关注标签:  [Agent ×] [RAG ×] [+ 添加]            │
│  🚫 屏蔽标签:  [元宇宙 ×] [+ 添加]                  │
│                                                      │
│  [保存]                                              │
└──────────────────────────────────────────────────────┘
```

## 六、关键业务规则

| 规则 | 说明 |
|---|---|
| 每日精选条数 | 10 条（可设 5/10/20） |
| 摘要长度 | 3-5 行（强制 LLM 输出） |
| 分类 | 🔥 头条 / 🏢 商业 / 📚 论文 / 🛠 工程 / 💡 观点 |
| 质量分 | LLM 评 1-5 分（用户可调权重） |
| 推送时间 | 默认 8:00（用户可改） |
| 邮件触发 | 每天 8:00 cron + 立即发送（重试 3 次） |
| macOS notification | 可选，触发即发 |
| 信源 | 默认 4 个（量子位 / 36氪 / HF / arXiv），可加 |
| 屏蔽 | "🔇 不再推类似" → 该 topic 7 天 -50% 权重 |
| Obsidian 写回 | 用户授权时写，路径 `~/Obsidian/coding/ai/` |
| 不调用其他模块 | 严格独立，**不查**学习复习 / 面试 / 知识库 |
| Profile 字段 | `digest_stats`（已读 / 收藏 / 阅读时长） |

## 七、边缘场景

| 场景 | 行为 |
|---|---|
| RSS 抓取失败 | toast 提示 + 用上一次缓存 |
| LLM 选题失败 | 退回到按时间排序前 10 条 |
| LLM 摘要失败 | 显示原文标题 + 链接（无摘要） |
| 邮件发送失败 | 重试 3 次（5 / 15 / 60 分钟） |
| macOS notification 失败 | 静默降级，不影响邮件 |
| 用户没邮箱 | 退化为站内 only |
| 用户屏蔽了 90% 的内容 | 提示"屏蔽过多，建议调关注标签" |
| 当天 0 条新内容 | 邮件写"今日无新内容"（不静默） |
| 周末推送 | 周六日仍然推送（用户可设"周末暂停"） |
| 时区变化 | 用户设置时区，推送按本地时区 |

## 八、成功指标

| 指标 | 目标 | 测量方式 |
|---|---|---|
| **每日读完率** | > 70% | 阅读完整日报的用户 / 收到推送用户 |
| **每日阅读时长** | 5-10 分钟 | 用户从打开到"全部已读"的时长 |
| **收藏率** | > 15% | 收藏条数 / 阅读条数 |
| **周报打开率** | > 80% | 周报邮件打开数 / 周报推送数 |
| **月报完成率** | > 60% | 月报读完用户 / 月报推送用户 |
| **屏蔽率** | < 5% | 屏蔽内容数 / 总推送数 |
| **推送不被屏蔽率** | < 5% | 用户设"不再接收"的比例 |
| **Obsidian 写回率** | > 20% | 写回次数 / 推送次数 |
| **用户留存** | > 60%（30 天） | 30 天后仍订阅 |

## 九、不在本设计内

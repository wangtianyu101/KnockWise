## 一、URL 路由

| 路径 | 功能 |
|---|---|
| `/push` | 总览（今日日报 + 本周周报 + 本月月报 tab） |
| `/push/daily/[date]` | 某天日报详情（如 `/push/daily/2026-06-22`） |
| `/push/weekly/[week]` | 某周周报详情（如 `/push/weekly/2026-W25`） |
| `/push/monthly/[month]` | 某月月报详情（如 `/push/monthly/2026-06`） |
| `/push/sources` | 信源管理（启停 RSS 源） |
| `/push/settings` | 推送设置（时间 / 渠道 / 标签） |
| `/push/bookmarks` | 我的收藏（跨日 / 周 / 月） |

#### 关键交互

| 交互 | 触发 | 行为 |
|---|---|---|
| tab 切换 | click | 切日报/周报/月报/信源/设置 |
| 收藏 ⭐ | click | 乐观更新，调 `POST /api/digest/bookmarks` |
| 分享 🔗 | click | 弹分享弹层（链接 / 复制） |
| 不再推类似 🔇 | click | 弹"为什么不喜欢"选择，调 `POST /api/digest/hide` |
| "全部已读" | click | 调 `POST /api/digest/read-all` |
| 写回 Obsidian | click | 调 `POST /api/digest/sync-to-obsidian` |
| 进度条 hover | hover | tooltip 显示每条状态 |

#### API 端点

| 时机 | 端点 |
|---|---|
| 页面挂载 | `GET /api/digest/dailies?limit=7`（最近 7 天列表） |
| 页面挂载 | `GET /api/digest/daily/latest?date=today`（今日详情） |
| 页面挂载 | `GET /api/digest/weekly/latest`（本周） |
| 页面挂载 | `GET /api/digest/monthly/latest`（本月） |
| 收藏 | `POST /api/digest/bookmarks` |
| 不再推类似 | `POST /api/digest/hide` |

## 三、关键 UX 细节

### 3.1 摘要质量（核心卖点）

| ❌ 差 | ✅ 好 |
|---|---|
| 标题：GPT-5 发布 | 标题 + 摘要：相比 4o 提升 3 倍上下文（128K→2M），新增视觉。30$/月定价引发争议。 |
| 没数字 | 关键数字必有（提升 X% / 估值 Y 亿 / 论文 Z 引用） |
| 没影响 | 必带"对谁有影响"（开发者 / 投资人 / 产品经理） |
| 太长 | 3-5 行强制（LLM prompt 控制） |

### 3.2 阅读进度

- 顶部进度条 `N/10 已读`
- 已读行变灰 + ✅
- 全部读完 → "🎉 今日读完，明天 8:00 见"
- 进度数据存 `digest_read` 表

### 3.3 推送时机（智能）

```
用户时区 (默认 Asia/Shanghai)
    ↓
cron 每分钟检查
    ↓
现在是用户设置的推送时间（默认 8:00）?
    ↓ 是
今天已推送过?（查 digest_read.pushed_at）
    ↓ 否
抓 RSS → LLM 选题 → 写 DB → 发邮件 → macOS notification
    ↓
记录 pushed_at
```

### 3.4 "🔇 不再推类似" 智能处理

- 弹层让用户选原因（"不感兴趣"/"质量低"/"内容已看过"）
- LLM 提取 topic 关键词
- 该 topic 在 7 天内 -50% 权重
- 7 天后自动恢复

## 五、状态机总览

### 5.1 日报阅读流

```
[进 /push] → 加载今日日报
                │
                ├─ 已读部分 → 灰色显示
                └─ 未读部分 → 正常显示
                          │
                          ▼ 用户点标题展开
                       [展开摘要]
                          │
                          ▼ 点"已读"
                       [调 POST /read] → 该条变灰
                          │
                          ▼ 全部读完
                       [🎉 toast: 今日读完]
```

### 5.2 推送流程

```
cron 8:00 (按用户时区)
    ↓
抓 RSS (4 源)
    ↓
合并去重 → 50+ 条
    ↓
LLM 选题（基于用户偏好 + 互动历史）
    ↓
10 条精选
    ↓
LLM 摘要（每条 3-5 行）
    ↓
分类排版
    ↓
写 digest_daily 表
    ↓
触发邮件（用户开启了的话）
    ↓
触发 macOS notification（用户开启了的话）
    ↓
记 digest_read.pushed_at = now
```

### 5.3 屏蔽处理

```
用户点 [🔇 不再推类似]
    ↓
弹层（"为什么不感兴趣"）
    ↓
用户选原因（不感兴趣 / 质量低 / 已看过）
    ↓
调 POST /api/digest/hide {qid, reason}
    ↓
LLM 提取 topic 关键词
    ↓
写 digest_hide 表
    ↓
7 天内选题 -50% 权重
    ↓
7 天后自动恢复
```

## 六、错误状态展示

| 场景 | 展示 |
|---|---|
| RSS 抓取失败 | toast "信源 X 抓取失败，已用缓存" |
| LLM 选题失败 | 退回到按时间前 10 条 + 标记 "fallback 模式" |
| LLM 摘要失败 | 显示原文标题 + 链接，标 "无摘要" |
| 邮件发送失败 | 重试 3 次（5/15/60 分钟），最终失败标红 |
| macOS 通知失败 | 静默降级 |
| 整条流程失败 | 当天日报空，显示 "今日无新内容" + 排查入口 |

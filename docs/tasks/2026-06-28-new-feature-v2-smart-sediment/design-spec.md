## 1. 用户旅程（必填）

### 完整流程（从用户视角）

```
场景 1：答完题看到沉淀
1. 用户在 /learn/[qid] 答完第 3 道题 → 提交 → 看到 AI 评分（4/5）
2. （1s 后，无感知）Profile weak_topics 自动多一项
3. （1s 后，无感知）~/Obsidian/coding/learning/2026-06-28.md 自动多一段
4. 用户点"下一题" → 继续答题（不感知沉淀发生）

场景 2：打开 Dashboard 看总结
1. 用户早上 9 点打开 /dashboard
2. 看到顶部新"今日学习总结"卡片：
   "昨天你答了 8 道题，掌握 2 个新 topic（React Hooks / TypeScript 泛型）
    弱项从「网络层」调整为「状态管理」"
3. 点击卡片 → 跳转 /profile 看详情

场景 3：周末看 Profile 趋势
1. 用户周六打开 /profile（新页）
2. 看到 4 个区块：
   - 顶部：累计统计（已做 N 题 / 已掌握 M / 学习中 K）
   - 左：weak_topics 列表（5 项，每项带题目链接 + 错题率）
   - 右：mastered_topics 列表（最近 10 个）
   - 底：learning_trajectory 趋势图（最近 12 周）
3. 点击 weak_topics 中"网络层" → 跳转 /learn 看相关题

场景 4：看 Obsidian 沉淀
1. 用户打开 /knowledge → 切到 stats tab
2. 看到"最近学习沉淀"卡片：
   "本周生成 5 个文件，最新：learning/2026-06-28.md"
3. 点击文件名 → 跳到 Obsidian 文件查看
```

### 关键决策点

- **决策 1**：Profile 趋势图用 recharts 还是手写 SVG？
  → **推荐 recharts**（V1 dashboard 已用，0 新增依赖）
- **决策 2**：Dashboard 总结卡是顶部一行（紧凑）还是大块（占 1/3）？
  → **推荐顶部一行**（不挤压 4 模块入口卡）
- **决策 3**：Obsidian 沉淀卡放 stats tab 还是新建 tab？
  → **推荐 stats tab 内**（不开新 tab，沉淀属于"统计"语义）

### 异常路径

- **Profile 字段全空**（新用户）→ 显示空状态："答 3 道题后这里会出现你的成长轨迹"
- **Obsidian vault 不存在**（开发机）→ Dashboard 卡显示："已生成 5 份沉淀笔记（路径：~/Obsidian/coding/learning/）"
- **LLM 调用失败** → Dashboard 总结卡显示"今日总结生成中…" + 重试按钮
- **learning_trajectory 数据不足 2 周** → 趋势图显示"继续学习 2 周后看到趋势"

## 4. 交互细节（必填 · 状态机）

### 4.1 `/profile` 关键状态表

| 状态 | 触发 | 视觉反馈 | 用户可操作 |
|---|---|---|---|
| **加载中** | 进入页面 | 4 个 Stat 卡片显示 skeleton + 趋势图占位 spinner | 等 |
| **数据为空** | 新用户 / 答 0 题 | 弱项/已掌握卡显示 "答 3 道题后这里会出现你的成长轨迹" 空状态插画 | 点"开始学" → /learn |
| **正常** | 有数据 | 4 块完整显示 + 趋势图 12 周数据点 | 点弱项 → /learn；点刷新按钮 |
| **刷新中** | 点"触发刷新画像" | 按钮变 loading；"刷新成功" toast | 不可重复点 |
| **刷新失败** | ProfileSettlementService 抛错 | toast "刷新失败，请重试" + 按钮恢复 | 点重试 |
| **学习趋势数据 < 2 周** | 新用户 | 趋势图显示 "继续学习 2 周后看到趋势" 占位 | — |

### 4.2 `/dashboard` 总结卡关键状态表

| 状态 | 触发 | 视觉反馈 | 用户可操作 |
|---|---|---|---|
| **加载中** | 页面 fetch 时 | 卡片显示 skeleton | 等 |
| **正常** | SummaryService.daily 成功 | 完整文本（昨天 N 题 / 弱项变化） | 点"查看画像" → /profile |
| **LLM 失败** | SummaryService 降级 | 卡片显示 "今日总结生成中…" + 旋转图标 | 30s 自动重试 1 次 |
| **完全失败**（3 次重试后） | LLM 不可用 | 卡片显示 "今日总结暂不可用" + 重试按钮 | 点重试 |
| **首次使用** | 用户 0 数据 | 卡片显示 "完成首日学习后，这里会显示你的成长总结" | — |

### 4.3 `/knowledge` 沉淀卡关键状态表

| 状态 | 触发 | 视觉反馈 | 用户可操作 |
|---|---|---|---|
| **正常** | 有生成文件 | 列表 5 个最新文件 | 点文件名 → 新窗口 |
| **空** | 0 个文件 | "答完第一道题后，这里会生成你的第一份学习笔记" | — |
| **Obsidian vault 不存在** | `_write` 返回 None | 卡片显示 "Obsidian 路径不存在，请在 ~/Obsidian/coding/ 创建目录" 警告条 | — |

### 4.4 异常流程

- **网络断开** → /profile / dashboard / knowledge 都显示顶部"网络错误"重试条
- **API 504**（LLM 超时）→ 总结卡降级到骨架态，30s 后自动重试
- **API 422**（参数错）→ 隐藏卡，显示"内容加载失败"灰条（不弹 toast，沉淀是软功能）

## 5. 视觉规范（必填）

### 5.1 颜色

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 | `#6366f1` (indigo-500) | V1 既有色，按钮、激活态 |
| 渐变 | `from-indigo-400 to-purple-400` | V1 既有，标题字 |
| 沉淀卡背景 | `bg-gradient-to-br from-indigo-500/10 to-purple-500/10` | V2 新增，让总结卡有"AI 生成"质感 |
| 弱项（红色） | `#f87171` (red-400) | V2 新增，弱项视觉提示 |
| 已掌握（绿色） | `#34d399` (emerald-400) | V2 新增 |
| 趋势图线 | `#a78bfa` (violet-400) | V2 新增 |
| 警告（Obsidian 不存在） | `#fbbf24` (amber-400) | V2 新增 |
| 文字主色 | `#f1f5f9` | V1 既有 |
| 文字辅助 | `#8b8fa3` | V1 既有 |
| 卡片背景 | `bg-[#0c1024]/80` | V1 既有 |
| 页面背景 | `bg-[#050914]` | V1 既有 |

### 5.2 字体

- 标题（页面 H1）：24px / 700 字重（V1 已有）
- 副标题（卡片标题）：16px / 600 字重（V1 已有）
- 统计数字（大数字）：36px / 700 字重（V2 新增，Stat 卡片用）
- 正文：14px / 400 字重（V1 已有）
- 辅助：12px / 400 字重（V1 已有）
- 行高：1.5（V1 既有）

### 5.3 间距

- 卡片内边距：24px（V1 既有）
- 卡片之间：16px（V1 既有）
- 区块之间：32px（V1 既有）
- Stat 数字与单位：4px（V2 新增）

### 5.4 组件库

- UI 库：**Ant Design 5**（V1 已用，dark theme）
- 图标：**@ant-design/icons**（V1 已用）
- 图表：**recharts**（V1 dashboard 已用，0 新增依赖）
- 骨架屏：**antd Skeleton**（V1 已用）
- Toast：**antd message**（V1 已用）

### 5.5 圆角 / 阴影

- 卡片圆角：12px（V1 既有）
- 按钮圆角：8px（V1 既有）
- 趋势图卡片阴影：`0 4px 16px rgba(99, 102, 241, 0.1)`（V2 新增，紫色微光）
